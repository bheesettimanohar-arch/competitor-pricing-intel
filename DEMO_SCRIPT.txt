[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture.
First, all incoming queries pass through the Security Checkpoint node to check for prompt injections and scrub private customer data.
If clear, the query goes to the orchestrator_agent.
The orchestrator_agent uses specialized sub-agents wrapped as AgentTools: the competitor_scraper_agent to collect prices and the pricing_analyst_agent to run the margin checks.
These sub-agents connect to a local MCP server that fetches pricing, cost, shipping, and inventory data via secure stdio tools.
Finally, the Check Approval node evaluates the price change. If the adjustment is within twenty percent, it finalizes automatically. If it exceeds that limit, it pauses at the human_approval_node for manager review.
